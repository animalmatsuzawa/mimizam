# 音声指紋生成の詳細アルゴリズム

本ドキュメントでは、mimizamの詳細アルゴリズムと設計について記します。

---

## 1. 音声指紋生成の概要

音声指紋システムでは、音声データから特徴的なパターンを抽出し、コンパクトなハッシュ値として格納します。このプロセスは以下の段階で構成されます：

### 基本処理フロー
1. **音声前処理**: サンプリングレート変換・正規化
2. **スペクトログラム生成**: STFT（短時間フーリエ変換）による周波数解析
3. **ピーク検出**: 局所最大値の抽出
4. **ハッシュ生成**: アンカー・ターゲットペアからのハッシュ値作成

mimizamでは音声特性に応じてパラメータを最適化し、高精度かつ高速な処理を実現しています。

---

## 2. 詳細アルゴリズム

### (1) スペクトログラム生成
音声データを時間-周波数表現に変換します。

- **STFT（短時間フーリエ変換）**: 音声信号を小さな時間窓に分割し、各窓でフーリエ変換を実行
- **FFTサイズ**: 通常1024または2048サンプル（周波数分解能と時間分解能のトレードオフ）
- **ホップ長**: FFTサイズの1/4程度（時間分解能の調整）
- **窓関数**: ハニング窓またはハミング窓（スペクトル漏れの抑制）

**設計根拠:**
- Wang 2003の基本手法に基づく時間-周波数解析
- 音声特性に応じたパラメータ調整により精度向上

### (2) ピーク検出
スペクトログラムから局所最大値（ピーク）を検出します。

- **局所最大値検出**: 近傍の周波数・時間範囲内で最大となる点を抽出
- **近傍サイズ**: 通常3×3または5×5の範囲（ノイズ除去と精度のバランス）
- **閾値設定**: 動的または固定閾値によるノイズ除去
- **密度制御**: 過密なピークの間引き処理

**最適化ポイント:**
- 音声の複雑度やノイズレベルに応じた閾値調整
- ジャンル特性を考慮したピーク密度制御

### (3) ハッシュ生成
検出されたピークからハッシュ値を生成します。

- **アンカー・ターゲット方式**: 各ピークをアンカーとし、周辺のピークをターゲットとして組み合わせ
- **ターゲットゾーン**: アンカーから一定範囲内のピークを選択
- **特徴量**: 周波数差・時間差を組み合わせた特徴ベクトル
- **ハッシュ化**: SHA-256などによる固定長ハッシュ値の生成

**技術的詳細:**
- ターゲットゾーンサイズの音声特性に応じた調整
- 量子化による特徴量の正規化

### (4) 実装における最適化

mimizamでは、より効率的で精度の高い指紋生成のために以下の最適化を実装しています：

#### 音声特性に応じたパラメータ調整
- **動的閾値**: 音声のノイズレベルに応じた検出閾値の自動調整
- **パラメータ選択**: 音声の複雑度やジャンルに基づく最適パラメータの選択
- **リアルタイム調整**: 処理中の音声特性変化への対応

#### 処理効率の向上
- **並列処理**: 複数の音声セグメントの同時処理
- **メモリ最適化**: 効率的なデータ構造の使用
- **段階的処理**: 必要に応じた詳細解析の実行
